1. Environment Setup

These commands prepare RStudio with the tools needed to work with air quality data.

  • install.packages(c("openair","tidyverse","lares")): Downloads the libraries from the internet.
  • library(tidyverse): Loads a group of tools used for cleaning and organizing data.
  • library(dplyr): Loads a part of Tidyverse used to modify and organize data tables.
  • library(lares): Library used to replace text and clean messy strings.
  • library(openair): Package designed for air quality and wind data analysis.

2. Data Import and Reshaping

Air pollution data often comes in a wide format (one column for each hour). These commands change it into a long format (one row for each observation) which is needed to create plots.

  • city <- read.csv("..."): Downloads the CSV file and saves it in a variable called city.
  • pivot_longer(city, cols=c(h01...h24), names_to="hour", values_to="value"): Changes the 24 hourly columns into two columns: hour and value.
  • city2 <- city1[-c(1,2,4,6:16)]: Removes columns that are not needed to keep the dataset cleaner.

3. Date and Time Formatting

Computers treat dates as text unless we convert them into real time objects. This step is very important for time-series analysis.

  • mutate(date = paste0(data, " ", hour)): Creates a new column that joins the date and the hour.
  • replaceall(city2, ...): Cleans messy time text and standardizes the time format.
  • as.POSIXct(city2$date, ..., tz="Europe/Madrid"): Converts the date column into a proper datetime format.
  • as.factor(city2$pollutant): Changes pollutant names into categories so they can be grouped in plots.

4. Advanced Visualization (openair)

These functions automatically create environmental charts.

  • timeVariation(city2): Shows how pollution changes by hour of the day, day of the week, and month.
  • trendLevel(city2, pollutant="H2S"): Creates a heatmap to see pollution trends over time.
